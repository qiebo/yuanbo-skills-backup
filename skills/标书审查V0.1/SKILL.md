# 标书审查（Bid / Tender Response Review）

针对中文招投标场景（公开招标 / 竞争性磋商 / 询价等），对「招标文件 + 一份或多份响应（投标）文件」做符合性与废标风险审查，产出结构化、可交付的审查报告。本技能由一次真实的多响应文件审查（竞争性磋商、3 份响应文件、65 万预算）沉淀而成，内置该次审查验证有效的并行子代理工作流与图片证据处理规则。

## 何时使用

- 用户给出 **1 个招标文件**（采购文件 / 磋商文件）和 **1~N 个响应文件**（投标 / 磋商响应），要求「检查 / 审查 / 核对 / 找问题 / 看会不会废标 / 符合性审查」。
- 适用产物：废标（无效响应）项矩阵、资格性与符合性核查、重点（★/▲）条款响应对比、分级问题清单（🔴废标风险 / 🟠严重 / 🟡一般 / 🟢建议）、页面友好的 HTML 报告。

## 核心工作流（6 步）

1. **抽取文本**：对每份 `.docx` 用 `scripts/extract_docx.py` 转 Markdown；`.doc` 用 `scripts/extract_doc_text.py`。输出到工作区 `output/`。
   - 脚本保留标题层级与表格，并在含图片处插入 `[IMAGE]` 标记——**这是关键信号**，提示该处证据以图片承载、文本层不可见（详见铁律 2）。
2. **提取审查基准**：精读招标文件文本，按 `references/review-baseline-template.md` 提取：项目信息、废标（无效响应）条款清单、资格性条件、符合性/形式要求、采购需求与重点（★/▲）条款、评分标准、偏离表要求。写入 `output/审查基准.md`。
3. **多份并行审查**：对 N 份响应文件，**每份启动一个独立子代理**（Agent 工具，general-purpose）并行审查，对照审查基准产出逐份报告（`output/子代理报告_XX.md`）。子代理提示词模板见下方「子代理提示词模板」。
4. **终审汇总**：主代理读取所有子代理报告，交叉核对废标项触及情况，去重定级，按 `references/report-template.md` 产出 `output/终审汇总报告.md`。
5. **HTML 呈现**：按 `assets/report-template.html` 的视觉风格（深色卡片、配色矩阵、分级问题卡片、三优先级整改清单）生成 `output/终审汇总报告.html`，页面友好、可打印。
6. **交付与说明**：present 报告，并明确「图片证据 / 签章原件无法由文本判定」的限制与免责。

## 三条铁律（来自真实审查经验，务必遵守）

- **铁律 1 — 废标优先**：先查废标（无效响应）条款，再查评分。废标项是零分风险，优先级高于任何得分点。
- **铁律 2 — 图片证据不可臆断**：docx 中营业执照、证书、信用截图、软件界面、合同、PPT、签字盖章等常以**图片**存在，文本抽取读不到。凡涉及图片证据的条款，判定写「**无法判定 / 需核对原件**」，**既不等于已满足也不等于未满足**；严禁因「文件里有这一章」就默认合规。
- **铁律 3 — 诚信红线**：重点核对「偏离表」声明与正文参数是否一致。若偏离表写「无偏离」但正文参数超出/低于招标要求（如速度上限超、处理器谱系不符），必须作为 **🟠 诚信风险** 单列，建议在磋商/澄清环节书面质询。

## 价格与一致性核查（通用，先做）

- 总报价 ≤ 最高限价（预算）；超限即废标。
- 报价明细表各行合计 = 报价一览表总价；不一致即废标（常见「格式四注」）。
- 报价有效期 ≥ 要求天数（如 45 天）。
- 不接受联合体时，不得出现联合体声明。
- 保证金：若要求而未提交 → 废标。

## 子代理提示词模板（复制后替换占位符）

> 你是一名严谨的招投标合规审查专家。独立审查响应文件 `{{响应文件文本路径}}`，对照审查基准 `{{审查基准路径}}`（含废标清单、资格/形式要求、重点条款、评分标准）。
> 步骤：1) 读审查基准建立标准；2) 完整读响应文件（图片处标注「需核对原件」，不得臆断已满足）；3) 按 7 项结构产出报告：
> ① 废标项逐条核查表（触及/未触及/无法判定 + 证据位置与说明）；
> ② 资格性与形式要求逐条核查；
> ③ 采购需求响应逐项核查（重点标 ★/▲，指出材料是否提供）；
> ④ 评分项覆盖度（指出可能得 0 分的项）；
> ⑤ 价格合规（≤限价、明细合计=一览表）；
> ⑥ 分级问题清单（🔴/🟠/🟡/🟢，每条含 位置 / 证据 / 风险 / 建议）；
> ⑦ 总体结论 + 截止前最紧急整改项。
> 要求：基于原文证据、不臆造；用 Write 写入 `{{输出路径}}`，回复摘要 ≤800 字。

## 详细审查清单

完整核查项（废标/资格/符合性/重点条款/技术/诚信/文档完整性/常见红旗）见 `references/checklist.md`，审查时逐项对照，避免遗漏。

## 局限与免责

- 本技能基于文本抽取，**不含图片 OCR / 视觉鉴伪**。最终废标与评分以磋商/评标小组对盖章正本及随附原件/图片的认定为准。
- 本报告为合规审查辅助意见，不构成法律意见。
