# 工作流：需求与证据

1. 读取全部用户材料，识别文类、交付格式、硬约束和禁止项。
2. 初始化项目目录：`python scripts/init_project.py --name 项目名 --type A --output 工作目录`。
3. 将用户原话拆入 `data/需求响应矩阵.csv`，不要用模型概括替代原始要求。
4. 将事实、数据、政策、价格、赛事和合作关系拆入 `data/事实证据台账.csv`。
5. 将无法确认的事项写入 `data/待确认事项.csv`，标明是否阻断交付。
6. 输出 `01_需求确认.md` 和 `02_调研与证据报告.md`。
7. 运行一次 `validate_project.py`，尽早暴露台账结构问题。
