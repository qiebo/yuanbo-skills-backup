# V1.0 → V2.0 迁移说明

## 不再沿用的做法

- 不再在 `generate_proposal.js` 中硬编码学校、预算和示例正文；
- 不再在找不到源文件时自动生成示例方案；
- 不再静默跳过 Markdown 表格；
- 不再维护静态“赛事已停办/仍在办”知识表；
- 不再把固定三方自评作为唯一质量门控；
- 不再把直接字符串替换 Word XML 作为标准修改方式；
- 不再仅通过读取前若干段文字判断 Word 质量。

## 新的内容源

| V1.0 | V2.0 |
|---|---|
| `0_需求确认.md` | `01_需求确认.md` + `data/需求响应矩阵.csv` |
| `1_调研报告.md` | `02_调研与证据报告.md` + `data/事实证据台账.csv` |
| `2_方案大纲.md` | `03_顶层设计决策.md` + `04_方案大纲.md` |
| `3_方案终稿.md` | `07_方案终稿.md` |
| JS 中的预算和表格 | `data/预算明细.csv` 等结构化台账 |
| 固定三方评审 | 硬校验 + 文类自适应评审 |
| Node docx 生成 | Pandoc + python-docx + 渲染 QA |

## 迁移步骤

1. 使用 `init_project.py` 初始化新项目；
2. 将 V1.0 的需求确认拆入需求响应矩阵；
3. 将调研结论拆入事实证据台账；
4. 将设备、课程、预算分别迁入对应 CSV；
5. 将 V1.0 终稿复制到 `07_方案终稿.md`；
6. 运行校验，逐项关闭错误；
7. 重新构建 Word 并逐页渲染检查。
