# 文档类型与结构规则

## A 类：自由建设方案

推荐结构：项目依据与现状 → 顶层设计 → 建设目标 → 建设内容 → 课程与应用 → 实施与验收 → 预算 → 成效与保障。

必须重点检查：学校专属性、课程设备空间映射、实施条件、运营机制。

## B 类：标准申报书

以官方字段为最高约束。不得为了追求“完整方案”擅自改字段顺序或增加大量不要求内容。

必须重点检查：字段覆盖、字数限制、政策口径、指标可量化、附件一致性。

## C 类：预算概算/配置清单

推荐结构：测算说明 → 分类清单 → 配置理由 → 价格依据 → 口径说明 → 风险与暂估项。

不强制概念词。必须重点检查：计算、单位、税率、价格有效期、数量逻辑、重复项。

## D 类：采购技术方案/合作协议

推荐结构：项目范围 → 技术响应 → 交付内容 → 实施计划 → 验收 → 培训/运维 → 责任边界 → 商务说明。

必须重点检查：参数逐条响应、交付边界、验收可执行、软硬件依赖、售后与数据安全。

## E 类：汇报稿/逐字稿

按汇报页序或决策逻辑组织：为什么做 → 做什么 → 与学校/政策的关系 → 如何实施 → 需要什么决策。

必须重点检查：口径统一、结论明确、数字可追溯、讲述时长和听众认知负担。

## F 类：课程建设方案

推荐结构：培养目标 → 学情与起点 → 课程结构 → 单元/项目任务 → 课时 → 资源 → 教学实施 → 评价 → 教师支持 → 迭代机制。

必须重点检查：目标—任务—评价一致，课时可行，设备和班额匹配，先修知识明确。
