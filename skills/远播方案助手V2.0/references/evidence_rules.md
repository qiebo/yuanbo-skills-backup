# 事实证据与真实性控制规则

## 1. 基本模型

任何进入方案的陈述都必须先判断属于哪一类：

| claim_type | 含义 | 典型语言 |
|---|---|---|
| verified_fact | 已由可靠来源核实的事实 | “学校现有……”“文件明确提出……” |
| user_provided | 用户或客户材料明确提供，但未独立核实 | “据学校提供资料……” |
| inference | 基于若干事实作出的分析判断 | “由此可见”“据此判断” |
| proposal | 本方案提出的建设动作 | “拟建设”“建议设置” |
| target | 可衡量的建设目标 | “计划在……前完成”“目标覆盖……” |
| expected_outcome | 建设完成后的预期结果 | “预期形成”“预计提升” |
| pending | 尚未确认 | 只能在内部稿中标注，不得伪装为事实 |

事实、方案、目标和预期成效不得混用时态。

## 2. 来源优先级

从高到低：

1. 法律法规、政府部门正式文件、官方数据库；
2. 学校、主办单位、厂商、合作单位的正式文件或官网；
3. 用户提供的合同、报价单、会议纪要、内部数据；
4. 权威研究报告、学术论文、行业协会资料；
5. 主流媒体的原始采访或公告转述；
6. 聚合网站、自媒体、论坛、搜索摘要。

高风险主张不得只依赖第 5—6 级来源。

## 3. 高风险主张

下列主张必须 100% 证据覆盖：

- “首个、唯一、率先、领先、填补空白、示范标杆”；
- 具体预算、价格、折扣、税率和采购口径；
- 设备型号、关键参数、认证和兼容性；
- 政策文件全称、文号、日期、适用范围；
- 赛事状态、名单属性、举办届次和学段；
- 学校荣誉、师资数量、课程数量、学生覆盖数；
- 高校、企业、专家的合作关系；
- 与升学、评价、招生、加分相关的表述；
- 安全、健康、隐私、数据合规相关承诺。

## 4. 证据台账字段

`data/事实证据台账.csv` 至少包含：

- `claim_id`：唯一编号；
- `claim`：准备写入方案的陈述；
- `claim_type`；
- `risk_level`：high/medium/low；
- `source_type`；
- `source_title`；
- `source_locator`：页码、表名、章节或 URL；
- `source_date`；
- `checked_at`；
- `valid_until`：有时效性时填写；
- `authority_level`：primary/secondary/tertiary；
- `confidence`：high/medium/low；
- `allowed_wording`；
- `target_sections`；
- `status`：verified/pending/rejected/expired。

## 5. 语言强度不得超过证据

| 证据情况 | 允许表述 | 不允许表述 |
|---|---|---|
| 官方材料直接证明 | “已建成”“现有”“明确提出” | 扩大适用范围 |
| 用户材料提供，未外部核实 | “据学校提供资料” | 当成公开核实事实 |
| 多条证据支持分析 | “据此判断”“有望形成” | “必然”“确定实现” |
| 项目设计动作 | “拟建设”“建议配置” | “已经具备” |
| 目标 | “计划达到”“目标覆盖” | “将确保”“一定实现” |
| 预期成效 | “预期形成”“预计提升” | 将预测写成既成事实 |

## 6. 时效性信息

政策、赛事、价格、产品规格、机构名称和合作关系均可能变化。使用时必须：

- 记录核验日期；
- 优先使用官方来源；
- 说明适用周期或有效期；
- 超过有效期时标记 `expired` 并重新核验；
- 不把“未列入当前名单”“本届未举办”和“永久停办”混为一谈。

## 7. 证据覆盖率

- 高风险事实：100%；
- 普通外部事实：不低于 95%；
- `pending` 高优先级事项：交付前必须为 0；
- `rejected` 与 `expired` 主张：不得进入终稿。

## 8. 引用呈现

对外版本不必把每句都写成学术论文，但必须保留可追溯性：

- 政策：正文写文件全称，必要时脚注；
- 数据：注明“据学校提供的××材料”；
- 预算：表中增加“测算依据/报价来源”；
- 参数：注明产品资料或技术文件；
- 高风险比较性结论：正文脚注或附录列明比较范围和检索依据；
- 内部版本：附《事实与证据索引》。
