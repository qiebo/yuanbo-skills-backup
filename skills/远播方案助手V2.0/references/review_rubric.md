# 行为锚定评审量表

## 1. 评分等级

| 等级 | 定义 |
|---:|---|
| 0 | 缺失、严重错误或不可用 |
| 1 | 有内容但存在关键缺口，不能实施/不能交付 |
| 2 | 基本成立，但需要实质性修改 |
| 3 | 可用，存在少量明确可修正问题 |
| 4 | 完整、证据充分、逻辑闭环，可直接交付 |

禁止只给总分。每个维度必须给出证据、问题位置、严重级别和修改建议。

## 2. 通用维度

- 需求覆盖；
- 事实与证据；
- 逻辑结构；
- 学校/客户专属性；
- 实施可行性；
- 预算与资源匹配；
- 指标与验收；
- 政策和风险；
- 语言与表达；
- 文档完整性。

## 3. 一票否决

以下任一存在即不通过：

- 虚构或来源不可追溯的高风险事实；
- 预算计算或口径错误；
- 具体政策、赛事、设备参数存在事实错误；
- 未授权使用合作单位、专家或品牌背书；
- 用户强制需求遗漏；
- 核心建设内容没有实施主体或验收标准；
- Word 丢失表格、正文、图片等核心内容；
- 文档存在明显占位符、待确认事项或排版缺陷。

## 4. 按文类选择评审角色

| 文类 | 必选评审 |
|---|---|
| 建设方案 | 教育方案、课程实施、工程/设备、政策合规 |
| 课程方案 | 课程、学科教学、评价、实施资源 |
| 预算/配置 | 采购、财务、技术适配、合规 |
| 采购技术方案 | 技术、实施交付、运维、安全/数据 |
| 汇报稿 | 战略表达、决策者视角、事实核验 |

## 5. 通过条件

- 所有必选维度 ≥3；
- 事实与证据、预算、需求覆盖三个维度必须为 4；
- 一票否决问题为 0；
- P1 问题全部关闭或由用户明确接受；
- P2 问题可记录为后续优化。
