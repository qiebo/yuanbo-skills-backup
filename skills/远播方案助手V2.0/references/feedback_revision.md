# 客户反馈与版本迭代

## 1. 提取顺序

1. Word 批注；
2. 修订记录；
3. 高亮文字；
4. 特定颜色文字；
5. 用户在消息中补充的修改要求。

颜色文字不必然是修改意见，必须结合上下文判断。

## 2. 修改台账

每条反馈记录：编号、原文位置、反馈内容、理解、修改动作、影响范围、状态、版本。

## 3. 修改原则

- 内容修改回到 Markdown 和数据表；
- 删除课程、设备、章节时检查数量、预算、映射、总述和附录；
- 修改预算时重新运行全部计算；
- 修改政策/赛事时更新证据台账；
- 修改后生成新版本，不覆盖上一版；
- 使用文字差异与页面渲染双重对比。

## 4. OOXML 使用边界

不得把字符串替换 `document.xml` 作为常规编辑流程。只有在批注、域、书签等上层库无法处理，且有备份、结构校验和渲染回归测试时，才允许定点修改 OOXML。
