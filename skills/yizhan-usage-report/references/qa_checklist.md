# QA 清单

每次交付前至少完成以下检查。

## 数据一致性

- Excel 目标学校名与网站后台学校名一致；如不一致，暂停并询问用户。
- 数据周期来自 Excel 月度行，不从报告模板文字推断。
- 学生账号总数来自网站学生管理分页总数。
- 累计交互总量按固定口径求和，不包含任何使用人数列。
- 本学期活跃为月度 `总使用人数` 最大值。
- 趋势表使用网站后台记录数趋势，报告中不得写作使用次数趋势。
- 漂流瓶覆盖学生如为发瓶学生去重，报告中必须标注该口径。
- AI 心语伙伴多口径不得混写。

## 报告内容

- 不出现 `{{placeholder}}`、`TODO`、`xxxx`、`###` 等调试文本。
- 高频问题进入正式报告前要过滤不适合公开呈现的敏感或不雅内容。
- 学校行动建议必须保留，不能只列数据。
- 结论要能从登记表追溯，不得用未获取字段补结论。

## 文件质量

- HTML 能离线打开，核心指标、四个图表、四个模块卡片可见。
- PDF 文件头为 `%PDF`，文件大小非 0，页数大于 1。
- Word 使用 `微软雅黑`，表头深蓝底白色加粗，表格内容垂直居中，关键数据分色加粗。
- 数据登记表至少包含：说明、核心指标、模块指标、网站记录趋势、内容与年级统计、模板字段路由、QA 与未决项。

## 推荐命令

```bash
python scripts/qa_report_bundle.py \
  --registry output/学校-生涯翼站报告数据汇总.json \
  --html output/学校-生涯翼站使用情况学期分析报告.html \
  --pdf output/学校-生涯翼站使用情况学期分析报告.pdf \
  --docx output/学校-生涯翼站使用情况学期分析报告.docx
```

如目标 agent 有 OfficeCLI，再运行：

```bash
officecli validate assets/templates/yizhan-report-template.docx
officecli view output/学校-生涯翼站使用情况学期分析报告.docx stats
```
