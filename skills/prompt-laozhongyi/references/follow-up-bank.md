# Follow-up Bank

Use these question patterns to clarify one dimension at a time. Do not ask every question. Pick the next smallest question that removes the biggest ambiguity.

## Goal

- What concrete deliverable should the model produce?
- Who is the audience for this output?
- What decision or action should this output support?
- Which part matters most: speed, completeness, accuracy, tone, or persuasion?
- What would make you say "this is done"?

## Output and acceptance

- What format should the answer use?
- How long or short should it be?
- What sections or fields are mandatory?
- What must be included?
- What must be excluded?
- How will you judge whether the answer is good enough?

## Inputs

- What materials will the model receive?
- Which inputs are authoritative?
- Is any essential information still missing?
- Should the model use only provided materials, or may it infer and generalize?
- Are there examples that show the desired result shape?

## Exceptions and boundaries

- If information is missing, should the model ask, flag, or continue with placeholders?
- If sources conflict, which source wins?
- What topics are sensitive, uncertain, or off-limits?
- What should the model never fabricate?
- What risk boundary matters here: compliance, confidentiality, customer impact, or executive visibility?

## Process

- Does this task need to be split into stages?
- Should the model analyze first and draft second?
- Should it check for gaps before generating the final output?
- Should it self-review against a checklist before finishing?
- Is there a preferred sequence that should not be skipped?

## Human and AI split

- Which parts can the model draft autonomously?
- Which parts require human judgment or approval?
- Should the model stop before final decisions and hand back a recommendation?
- What should always remain a human call?

## Skill potential

- Is this task repeated often?
- Is the workflow stable across runs?
- Can the required inputs be defined clearly?
- Can the acceptance criteria be standardized?
- Would a reusable skill save time or reduce variance here?

## Escalation patterns for vague answers

If the user stays abstract, narrow the question:

- Replace "What style?" with "Should this read like a status update, an analysis memo, or a client-ready draft?"
- Replace "What output?" with "Do you want bullets, a table, or a polished narrative?"
- Replace "What matters most?" with "If forced to trade off, should the model favor completeness or speed?"
- Replace "Any constraints?" with "Are there length, confidentiality, or approval constraints?"
