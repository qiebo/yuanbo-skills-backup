# 资料组织法（内容严谨、有出处）

> 目标：使方案内容严格依据用户提供的资料组织，杜绝凭空发挥。
> 配合 `writing_guide.md` 的"反对虚构"红线使用。进入阶段二（调研）与阶段四（撰写）前阅读。

---

## 一、核心原则

1. **每一句事实都有来源**：政策表述、设备型号、预算数字、课时、赛事出口，必须来自用户提供的资料或已校验的公开信息。
2. **资料缺失即追问或标注"待确认"**，绝不拍脑袋补全。
3. **资料是骨架不是填充**：先按资料搭结构，再行文；不先写空壳再找资料硬塞。
4. **保留溯源线索**：调研报告中记录每条信息的出处链接，撰写时直接引用，便于用户事后查证。

---

## 二、资料分类与章节映射

| 资料类型 | 典型形态 | 落位章节 | 使用要求 |
|----------|----------|----------|----------|
| 政策文件 | 教育部/省市通知、白皮书 | 背景段政策锚点 | 准确到文件全称；附原文链接；用当期有效表述 |
| 地方/产业资料 | 区域规划、产业报告 | 背景段地方语境 | 点明与学校地缘的关联，不做空泛铺垫 |
| 学校资料 | 校史文脉、校训精神、建设优势（场地/师资/设备/课程/合作）、未来发展规划、会议纪要、需求描述 | 基因钩子＋痛点缺口＋已有基础 | 四维综合提炼专属词汇；会议纪要中的诉求必须回应；基因须有调研证据，禁止凭印象 |
| 供应商报价单/设备清单 | xlsx 报价、产品手册 | 建设内容（设备/功能）＋预算 | 设备写型号与功能定位，不写参数说明书；预算数字以报价单为准 |
| 高校/合作资源 | 合作协议、专家名单 | 顶层设计/保障 | 如实写入，不夸大合作深度 |

---

## 三、标准组织顺序（撰写阶段遵循）

```
背景段  = 政策（文件） → 地方产业（资料） → 学校基因（校史文脉/校训精神/建设优势/未来方向） → 痛点缺口（会议纪要/需求）
顶层设计 = 基于"学校基因 + 缺口"提炼概念词与结构（见 top_design_playbook.md）
建设内容 = 基于"报价单/设备清单"组织空间、装备、平台，写明功能定位
课程规划 = 基于"已有课程 + 需求"设计阶梯体系，标注真实课时/赛事出口
预算分配 = 基于"报价单"汇总，注明分配原则与出处
```

---

## 四、溯源标注实操

- 调研阶段：在 `1_调研报告.md` 每条结论后标注 `[来源：文件名 / 链接]`。
- 撰写阶段：政策类直接引文件名；设备类引"据供应商XX报价单"；预算类引报价单页码/表名。
- 不确定项：文中用"（待用户确认：XX）"标注，并在 `0_需求确认.md` 的待确认事项中登记，禁止写成确定事实。

## 五、常见失真与规避

| 失真类型 | 表现 | 规避 |
|----------|------|------|
| 虚构设备参数 | "64通道、采样率1000Hz"无来源 | 仅写"专业级XX设备，满足教学与科研需求"，细节待报价单 |
| 预算拍脑袋 | 自编金额与占比 | 严格以报价单汇总，占比据实计算 |
| 政策张冠李戴 | 引用已过期或不存在的文件 | 校验文件名与年份，附链接 |
| 校训贴错 | 套用他校校训 | 仅用用户提供的本校校训原文 |
| 需求漏回应 | 会议纪要提的痛点未在方案中解决 | 撰写前逐条核对会议纪要诉求清单 |
