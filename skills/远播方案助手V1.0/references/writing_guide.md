# 教育方案写作指南

## 一、去AI化核心原则

### 1.1 禁用表达清单

以下表达在教育方案中出现时，会立即暴露AI写作痕迹：

| AI痕迹表达 | 问题分析 | 推荐替代 |
|-----------|---------|---------|
| 一是...二是...三是... | 机械条目罗列 | 合并为连贯段落，用逻辑关系词连接 |
| 首先...其次...最后... | 空泛的流程标记时显刻板 | 若仅是空泛过渡则删除；若是**实质的三点设计聚焦**（如学军 X-Space 用"首先空间利用/其次学生获益/最后面向未来"承载真实设计逻辑），可保留为逻辑支架，但三点须都有干货、不注水 |
| 综上所述 | AI高频结尾语 | 直接收束论点，无需标签 |
| 值得注意的是 | 空洞强调 | 直接陈述事实 |
| 不难发现 | 预设读者判断 | 删除，直接给出结论 |
| 具有重要的现实意义和深远的历史意义 | 万能空话 | 具体说明对谁、在什么方面有意义 |
| 随着...的不断发展 | AI万能开头 | 直接切入具体背景 |
| 多维度、全方位、深层次 | 形容词堆砌 | 具体列举维度 |
| 为...提供有力支撑/坚实基础 | 模板化表述 | 具体说明支撑方式和效果 |
| 不是…而是…（否定—转折—肯定判断句） | AI高频对照结构，机械感强、转折生硬，极易暴露机器写作 | 直接陈述立场如"该项目是响应国家战略的必然选择"；或以"关键在于/理应"带出判断 |

### 1.2 条目罗列转段落示例

**原文（AI风格）**：
> 实验室建设具有以下三方面意义：一是提升学生科学素养，二是促进教师专业发展，三是推动学校特色发展。

**改写（专家风格）**：
> 实验室建设对学生科学素养的提升体现在真实项目情境中的探究能力培养；对教师而言，新的技术平台将推动教学方法的更新与跨学科教研的开展；从学校发展层面看，这一项目有助于形成科技教育特色品牌，增强学校在区域教育格局中的辨识度。

### 1.3 过渡语自然化

**生硬过渡**：
> 前文分析了项目背景，接下来将从建设目标角度进行阐述。

**自然过渡**：
> 基于上述政策背景与学校现状，建设目标的设定需要兼顾国家课程要求与校本特色发展两个层面。

## 二、教育方案语言风格

### 2.1 语气定位

- 客观陈述为主，避免推销感（"必将"、"定能"、"显著提升"）
- 用数据说话，减少主观评价
- 政策引用准确到文件名称和文号
- 对不确定内容使用审慎表述（"拟"、"计划"、"预期"）

### 2.2 句子节奏

- 长短句交替，避免连续短句造成的碎片化感
- 适当使用复合句体现逻辑层次
- 专业术语首次出现时简要说明，后续直接使用
- 段落长度控制在3-6行，重点段落可更长

### 2.3 数字与单位

- 预算金额统一用"万元"为单位，保留两位小数
- 设备数量与金额对齐，表格中右对齐
- 年份统一用四位数字，不用"今年""去年"等相对时间
- 百分比统一用阿拉伯数字加百分号

## 三、常见章节写作模板

### 3.1 项目背景

标准结构：政策背景 -> 区域/学校现状 -> 问题与需求 -> 建设必要性

示例开头：
> 2023年教育部等十八部门联合印发的《关于加强新时代中小学科学教育工作的意见》明确提出，要"强化实验教学，构建实践教学体系"。浙江省作为数字化改革先行区，在《浙江省教育事业发展"十四五"规划》中将人工智能教育纳入中小学课程体系建设重点。湖州中学作为省一级重点中学，现有科技教育基础良好，但在脑机接口等前沿技术领域的实验条件与课程体系尚属空白，难以满足学生拔尖创新人才培养的差异化需求。

### 3.2 建设目标

避免：
> 总体目标：建成国内一流、省内领先的脑机接口实验室。

推荐：
> 建设周期内，实验室将承担面向高一至高二年级的选修课程教学、学生科技创新项目指导以及区域教师培训三项核心功能。预期建成后可同时容纳30名学生开展实验活动，年服务学生规模达到200人次以上。

### 3.3 经费预算

预算说明写法：
> 项目总预算310万元，其中设备购置费占比78%（242万元），装修改造费占比12%（37万元），课程开发与师资培训费占比10%（31万元）。资金来源包括市级教育专项拨款220万元（占71%），学校自筹90万元（占29%）。

### 3.4 预期成效

避免罗列具体赛项：
> 学生将参加全国青少年科技创新大赛、机器人竞赛等赛事。

推荐：
> 学生科技创新项目预期在省级及以上青少年科技竞赛中获得奖项，相关研究成果拟整理为校本课程案例集。

## 四、方案类型与篇幅控制

| 方案类型 | 建议篇幅 | 技术细节深度 | 重点章节 |
|---------|---------|-------------|---------|
| 预算申报方案 | 8000-12000字 | 方向性描述，不写实现细节 | 建设内容、预算 |
| 项目建设方案 | 15000-25000字 | 可展开技术路线与实施步骤 | 技术方案、实施计划 |
| 课程建设方案 | 10000-15000字 | 侧重教学目标与内容架构 | 课程设计、评价体系 |
| 综合申报方案 | 12000-18000字 | 平衡深度与广度 | 背景、方案、成效 |

## 五、文档迭代工作流

### 5.1 阶段文档结构

每个项目在工作目录中维护以下Markdown文件（以"湖州中学脑机接口实验室"为例）：

```
e:/codex/湖州中学/
├── 0_需求确认.md          # 阶段一产出
├── 1_调研报告.md          # 阶段二产出
├── 2_方案大纲.md          # 阶段三产出（后续在此填充正文）
├── 3_方案终稿.md          # 阶段四/五产出（完整正文源文件）
├── 三方评审汇总.md         # 评审门控产物
├── 评审报告_课程评审员.md   # 评审员1
├── 评审报告_商业评审员.md   # 评审员2
├── 评审报告_政策评审员.md   # 评审员3
└── 湖州中学脑机接口实验室建设项目申报方案.docx   # Word交付物
```

### 5.2 迭代修改操作

**场景1：用户要求"去AI味儿"**
1. 打开 `3_方案终稿.md`
2. 定位到条目罗列的段落
3. 改写为自然段落叙述
4. 保存后重新运行 `generate_proposal.js` 生成Word

**场景2：用户要求"删掉3.4节"**
1. 打开 `2_方案大纲.md`，删除对应章节
2. 打开 `3_方案终稿.md`，删除对应章节正文
3. 保存后重新生成Word

**场景3：用户要求"补充政策依据"**
1. 打开 `1_调研报告.md`，补充新政策内容
2. 打开 `3_方案终稿.md`，在对应章节引用新政策
3. 保存后重新生成Word

**场景4：用户要求"调整预算数字"**
1. 打开 `3_方案终稿.md`，找到预算表格对应的Markdown表格
2. 修改数字
3. 保存后重新生成Word（脚本中的表格数据同步更新）

**场景5：用户提交标注修改意见的Word文档**
1. 解包docx文件，提取所有红色标注文本
2. 整理为编号修改意见清单
3. 向用户确认理解是否正确
4. 逐条在XML中修改（注意闭合标签完整性）
5. 重新打包验证，生成新版本

### 5.3 Markdown转Word的注意事项

- Markdown中的表格语法（`| 列1 | 列2 |`）在生成Word时需手动转换为 `makeTable()` 调用
- Markdown中的标题（`## 标题`）对应脚本中的 `h1()` / `h2()` / `h3()` 调用
- Markdown中的普通段落直接作为 `p()` 的参数
- 图片、复杂排版在Markdown中用占位符标注，在脚本中单独处理

## 七、老师反馈修改规范

### 7.1 Word XML修改操作要点

当用户提交已标注红色修改意见的Word文档时：

1. **解包**：docx本质为ZIP，解包后编辑 `word/document.xml`
2. **定位红色标注**：搜索 `<w:color w:val="FF0000"/>` 定位所有修改意见
3. **跨run处理**：同一句话可能被拆分到多个 `<w:r>` 中（因字体/颜色不同），修改时需合并或扩大匹配范围
4. **XML闭合标签**：替换时**必须**保持 `</w:r>`、`</w:p>` 等闭合标签完整，否则pack时报错 `mismatched tag`
5. **精确匹配**：使用Python `repr()` 输出精确XML片段后复制到替换字符串，避免肉眼比对遗漏
6. **版本递增**：每次修改生成新版本（v1→v2→v3），保留历史

### 7.2 删除内容的连锁修改清单

删除某个课程/课题时，必须逐项检查并同步修改：

- [ ] 正文总述中的数量（"三个课题"→"两个课题"）
- [ ] 该课程的标题段落
- [ ] 该课程的描述段落
- [ ] 该课程配套的课程信息表格（适用对象/编程语言/课时/赛事）
- [ ] 竞赛体系中该课程的竞赛出口
- [ ] 预算表中该课程对应的费用项
- [ ] 其他章节对该课程的引用

### 7.3 常见XML修改错误与修复

| 错误现象 | 原因 | 修复方法 |
|---------|------|---------|
| Edit工具匹配失败 | XML中字符串与肉眼看到的不完全一致 | 用Python `repr()` 输出精确片段再匹配 |
| pack报错 `mismatched tag: line XXX` | 替换时误删闭合标签 `</w:r>` | 检查替换内容，补充 `</w:r>` |
| 修改后文本丢失 | 替换范围过大，覆盖了相邻run | 缩小替换范围，只替换目标run |
| 中文显示异常 | XML编码不一致 | 确保读写均使用UTF-8 |

## 九、金标准正例库（来自已交付方案，供模仿）

> 以下句子/段落摘录自经客户确认的最终交付版，代表"去AI化且正式规范"的真实达成水平。
> 模仿其**句式结构与信息密度**，而非照搬内容。所有事实性信息仍须以用户提供的资料为准。
> 仅收录**最终交付版**：工作草稿、"修改vN""老师意见-已修改""RFV0x"等迭代稿中的表述一律不计入，避免把未定稿甚至AI味残留误当金标准。

### 9.1 背景段：政策→地方→学校→痛点的四层一气呵成

> 在国家深入实施科教兴国战略、积极谋划"十五五"规划的宏大背景下，拔尖创新人才的自主培养已成为教育强国的核心议题。上海作为中国科技创新的排头兵，明确提出了集成电路、生物医药、人工智能三大先导产业协同发展的战略布局。位育中学作为上海市的窗口学校，始终走在科创教育的前列……然而，芯片作为AI时代的基础设施，其核心价值最终体现在对千行百业的赋能上。目前，学校在芯片"应用"环节的教育尚存空白。

**可学之处**：不堆"随着…发展"，直接从最高政策切入，落到该校独有缺口；用"然而"制造转折张力，而非罗列。

### 9.2 顶层设计概念句：概念词＋隐喻＋闭环

> 建设的核心逻辑在于解决"芯片造出来怎么用"的关键一环……构建起从"识芯（原理）""造芯（工艺）"到"用芯（应用）"的完整教育闭环。

> 从南货站的传统转运到未来智造的智能递送，从单一学科探究到跨场景的价值践行，系列课程升级构建起"识技—研技—用技"的完整教育闭环。

**可学之处**：用"动词链"呈现逻辑完整性，比"形成一体化体系"具体百倍。

### 9.3 具象隐喻句：把抽象技术讲出画面感

> 对于学生而言，这不仅仅是一个生物实验，更是一次"软件定义硬件"的工程实践。他们将学习如何通过编写代码来控制液滴的运动轨迹，像指挥交通一样指挥微观化学反应。

**可学之处**：用生活化比喻（指挥交通）解释微观操作，技术不枯燥；这是"高端但不晦涩"的关键技法。

### 9.4 校训融合句：把校训翻译成项目中的具体行为

> 诠释"诚"：引导学生在科学探究中尊重事实、严谨求证，在数据分析中忠于原始信息，在团队协作中真诚沟通。践行"勇"：鼓励学生勇敢面对复杂的工程挑战，不畏惧失败，在反复的测试与迭代中锤炼坚韧不拔的意志。

**可学之处**：校训不直接引用，而是拆成可观察的学生行为，融入课程环节。

### 9.5 收束句：不用"综上所述"也能利落收尾

> 学校既有的工程结构、STEM制造、地理DIS数据采集、CYPT探究与学术表达、Deep Racer算法训练等优势资源"串珠成链"，构建面向真实生活情境的跨学科科学学习工坊。

**可学之处**：以一个强隐喻收束段落，信息密度高，无标签化结尾。

### 9.6 学校专属词汇表（强烈建议）

金标准方案的共同特征：大量使用**该校独有词汇**，而非通用词。写作前务必向用户索取并贯穿使用：
- 校训原文与解释词（如"诚勇""适性""致远"）
- 学校已有课程/平台专有命名（如"两力八要素""储能·适性·致远三级课程图谱"）
- 地缘产业特征词（如"南货站文脉""物联网产业高地"）

> 替换通用词为专属词，是消除"换校名即可复用"AI感的最有效手段。

### 9.7 转译隐喻："把X变成Y"接入非典型主题

> 把前沿产业的真实课题引入中学课堂……打通学生的升学与职业发展闭环。（金华外国语学校）

> 把校园转化为可持续使用的学习场域。（进才中学）

**可学之处**：当项目主题不属于典型科创（如美育、戏剧、语言、空间改造），用"把A变成B""把X接入/转化为Y"的转译句式，把非典型主题锚定到科学教育主线上，既新颖又不突兀。
